# 결정 로그 (Decision Log)

> 자가개선 루프의 각 설계 결정과 "왜 그렇게 정했는가". 면접 방어용이자, AI 자동생성물과의 차별점.
> 설계 전문: `docs/active_learning_design.md`

| ID | 결정 | 값 | 이유 (도메인 근거) |
|----|------|----|------|
| D1 | 재학습 트리거 | 교정 라벨 **20건** 누적 OR 드리프트 alert, 먼저 오는 것 | 최신성을 자주 반영하려 임계값을 작게. 라인/입력 변화에 빠르게 대응. (`VLM_RETRAIN_LABEL_THRESHOLD`로 조정) |
| D2 | 재학습 데이터 구성 | 원본 train + 교정분(틀렸던 케이스 가중) | 교정분만 쓰면 기존 지식을 잊는 catastrophic forgetting. 원본 유지 + 새 실패 가중으로 균형. |
| D4 | 모델 승격 게이트 | 고정 평가셋에서 위험점수 ≤ 현행 **AND** 유형정확도 ≥ 현행 − 0.01 | 규제(제조/의료) 관점: 나쁜 모델 출고 방지가 최우선. 정확도 한 숫자가 아니라 "위험점수"로 안전 우선 판정. |
| D3 | 액티브러닝 샘플 선정 | confidence 낮은 것 우선 (uncertainty) — 기본값 | 단순·효과적. 검토 큐가 이미 이 방식. 다양성 샘플링은 추후. |
| D5 | 롤백 | 이전 어댑터 보관 + 수동 롤백 우선 | 자동 롤백은 오작동 위험. 직전 버전 항상 보관해 즉시 복구 가능하게. |
| D6 | 라벨 신뢰도 | 검토자 1인 + reviewer 기록 | 속도 우선. 복수 합의는 비용↑, 추후 확장. |
| D7 | 재학습 방식 | base에서 LoRA 재학습 (이어학습 ❌) | 재현성↑, 64분이라 감당 가능. 이어학습은 드리프트 누적 위험. |
| D8 | 데이터 거버넌스 | 레지스트리에 model_version → 학습데이터 ref → 평가결과 기록 | ISO식 추적성. "이 모델이 어떤 데이터로 나왔나" 추적 가능. |
